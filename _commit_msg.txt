v1.0.03: save.dat + game.db move to AppData so updates don't lose data

Before: save.dat and game.db lived in cwd (next to the .exe). Players
who downloaded a new .exe to a different folder lost their account
+ progress (still on disk, just not where the new build looked).

After: both files live in a per-user folder

    Windows  %APPDATA%\DieAgailTroll\
    macOS    ~/Library/Application Support/DieAgailTroll/
    Linux    ~/.local/share/DieAgailTroll/

First-run migration: if the AppData target doesn't exist yet but a
legacy file is sitting next to the .exe, copy it across. So a player
upgrading from v1.0.02 -> v1.0.03 keeps everything: account,
last_username, max_normal_level, max_nightmare_level, total_deaths,
all audio + display settings.

Implementation:

  systems/paths.py
    - user_data_dir() resolves the OS-appropriate AppData folder
    - migrate_legacy_file(name) returns the AppData path, copying
      from cwd on first use if needed

  systems/save_mgr._path()  -> migrate_legacy_file("save.dat")
  systems/auth_db._db_path() -> migrate_legacy_file("game.db")

  Env overrides DIEAGAIN_SAVE_PATH / DIEAGAIN_DB_PATH are still honoured
  first, so the smoke test continues to isolate its data in a tempdir
  without touching production.

Verified end-to-end via _migration_check.py: a fake v1.0.02-era save
in cwd is read correctly by v1.0.03, persisted to AppData, and remains
readable after the legacy file is deleted.

README + version stamp bumped to v1.0.03.
